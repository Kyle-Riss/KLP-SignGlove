## 🎯 프로젝트 배경 및 필요성

### 문제 정의
1. **의사소통 장벽**: 청각장애인과 일반인 간의 의사소통 어려움
2. **기존 솔루션의 한계**: 
   - 고가의 전문 장비 (수십만원 ~ 수백만원)
   - 복잡한 설치 및 사용법
   - 한국어 수어 지원 부족
3. **기술적 격차**: ASL(미국 수화) 대비 한국어 수어 인식 기술 부족

### 해결 방안
- **저비용 하드웨어**: Arduino + Flex 센서 조합 (5만원 이하)
- **실시간 인식**: 36Hz 샘플링으로 자연스러운 수화 인식
- **한국어 특화**: 한국어 수어 자음/모음 24개 클래스 지원
- **오픈소스**: 누구나 접근 가능한 오픈소스 솔루션

## 🔬 기술적 접근 방법

### 하드웨어 구성
```
┌─────────────────────────────────────┐
│           SignGlove 하드웨어        │
├─────────────────────────────────────┤
│ • Arduino Nano 33 IoT (마이크로컨트롤러) │
│ • SF15~150 Flex 센서 5개 (각 손가락)    │
│ • 내장 IMU 센서 (pitch, roll, yaw)     │
│ • 내장 WiFi/Bluetooth 모듈           │
│ • 배터리 및 충전 회로               │
└─────────────────────────────────────┘
```

### 소프트웨어 아키텍처
```
┌─────────────────────────────────────┐
│           실시간 인식 시스템        │
├─────────────────────────────────────┤
│ 1. 데이터 수집 (36Hz)               │
│ 2. 전처리 (노이즈 제거, 정규화)      │
│ 3. 특징 추출 (24개 특징)            │
│ 4. 모델 추론 (LSTM/GRU/Transformer) │
│ 5. 후처리 (연속성 검증)             │
│ 6. 텍스트 출력                      │
└─────────────────────────────────────┘
```

### 모델 구조
- **입력**: 24개 특징 (원본 8개 + 속도 8개 + 가속도 8개)
- **모델**: Bidirectional LSTM/GRU + Attention
- **출력**: 24개 클래스 (자음 14개 + 모음 10개)
- **성능 목표**: 95% 이상 정확도

## 📊 데이터 수집 계획

### 데이터셋 구성
| 구분 | 클래스 수 | 샘플 수 | 수집 방법 |
|------|-----------|---------|-----------|
| **자음** | 14개 | 420개 | 5명 × 3회 × 14개 |
| **모음** | 10개 | 300개 | 5명 × 3회 × 10개 |
| **총계** | 24개 | 720개 | 5명 × 3회 × 24개 |

### 수집 프로토콜
1. **참가자**: 5명 (다양한 손 크기, 연령대)
2. **수집 환경**: 일정한 조명, 배경
3. **수집 방법**: 
   - 초기 자세 유지 0.5초
   - 각 클래스별 3회 반복
   - 36Hz 샘플링으로 300프레임 수집
4. **품질 관리**: 실시간 데이터 검증 및 이상치 제거

### 현재 진행 상황
- 🔄 **예정**: ㄱ, ㄴ, ㄷ, ㄹ, ㅁ (5개 클래스)
- 📅 **계획**: 나머지 19개 클래스 수집
- 📅 **완료 예정**: 2025년 7월 말
